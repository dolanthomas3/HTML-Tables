HTML Tables

Structuring tabular data is a common task, and here we will consider all of the elements and attributes used most often for this purpose.

HTML Table Basics

Here we cover table basics, like rows and cells, headings, making cells span multiple columns and rows, and grouping together cells in a column for styling purposes.

What is a table?

A table is a structured set of data made up of rows and columns (tabular data) that allows for quick and easy lookup of values that indicate a connection between different types of data (a person and their age, a day of the week, or a timetable for a local swimming pool).

Table Styling

Readability is greatly enhanced by CSS styling; on the web, it is a must. Good structure with HTML is also a must, so make sure to use at least a good foundation.

When NOT to use HTML tables

HTML tables should never be used for a website layout, CSS styling should always be used instead. Some main reasons are:

  1. Layout tables reduces accessibility for visually impaired users: screen readers (used by blind people) interpret tags that exist in HTML pages and read out the contents to the user. This becomes confusing when table outputs are read by a screen reader over a whole website.

  2. Tables produce tag soup: table layouts are generally more involved in complex markup structures than proper layout techniques; this makes the overall code harder to write, interpret, and maintain.

  3. Tables are not automatically responsive: The use of proper layout containers (like <header>, <section>, <article>, and <div>) default widths are set to 100% of their parent elements. Tables are sized according to their content (by default) instead, so extra measures are needed to get table layout styling to effectively work across a variety of devices.

These are the elements and attributes that go into making a table:

  • <td>: The smallest container inside a table is a table cell, "td" means "table data". Putting these one after another lines the cells up into a row.

  • <tr>: Used to stop a row from growing, and start placing cells on a subsequent row; "table row";

  • <th>: "table headers"; a special cell that goes at the start of a row or column and defines the type of data that row or column contains; they allow you to make tables more accessible by associating each header with all the data in the same row or column. Screen readers are then able to read out a whole row or column of data at once.

  • colspan="#": allows cells to span columns; the # refers to the number of columns to span.

  • rowspan="#": allows cells to span rows; the # refers to the number of rows to span.

Styling without <col>

When NOT using <col>, styling has to be specified for each <th> or <td> in the table that needs the styling. This is time consuming and tedious. Here is an example:

Data 1 Data 2
Calcutta Orange

Styling with <col>

When using <col>, the styling information needs to be specified only once using the element. An example:
